## 🎯 段階的練習課題

### レベル1: 小さな機能追加（理解の確認）

#### 課題1-1: シンプルなクラスの作成
**目標**: 基本的なクラスとオブジェクトの概念を理解する

1. `Person`クラスを作成
   - フィールド: `name`（String）、`age`（int）
   - コンストラクタ: 名前と年齢を受け取る
   - メソッド: `introduce()` - 自己紹介を出力する

2. `Main`クラスで`Person`オブジェクトを作成し、`introduce()`を呼び出す

**チェックポイント**:
- クラスとオブジェクトの関係を理解できたか？
- コンストラクタの役割を理解できたか？
- メソッドの呼び出し方を理解できたか？

#### 課題1-2: Getter/Setterの追加
1. 課題1-1の`Person`クラスにGetter/Setterを追加
2. `name`と`age`を`private`にする
3. `Main`クラスでGetter/Setterを使って値を取得・設定する

**チェックポイント**:
- カプセル化の概念を理解できたか？
- なぜ`private`を使うのか説明できるか？

#### 課題1-3: メソッドのオーバーロード
1. `Person`クラスに、引数なしの`introduce()`メソッドを追加
2. 引数なしの場合は「こんにちは」だけを出力
3. 引数ありの場合は従来通り自己紹介を出力

**チェックポイント**:
- メソッドのオーバーロードを理解できたか？

## 📚 学習の進め方

### ステップ1: コードを声に出して説明する
既存のコードを、1行ずつ声に出して説明してみてください。
「この行は何をしているか？」「なぜこれが必要か？」を自分に問いかけます。

### ステップ2: 小さな変更を加える
まずは既存のコードを少しだけ変更してみます：
- メソッド名を変える
- 変数名を変える
- メッセージの文言を変える
- フィールドを追加する

### ステップ3: エラーを意図的に起こす
エラーを起こして、どう対処するか学びます：
- `private`を`public`に変えてどうなるか見る
- メソッドの戻り値の型を間違えてみる
- コンストラクタを削除してみる
- 例外処理を削除してみる

### ステップ4: ドキュメントを読む
公式ドキュメントやチュートリアルを読んで、概念を理解します：
- Oracle公式Javaチュートリアル
- Java APIドキュメント
- オブジェクト指向プログラミングの概念

### ステップ5: 練習課題に挑戦
上記の課題を順番に取り組みます。分からない時は：
1. まず自分で考えてみる（30分）
2. エラーメッセージを読む
3. 公式ドキュメントを調べる
4. それでも分からなければ、AIに「なぜこのエラーが出るのか」を聞く

## ✅ 理解度の確認方法

以下のことができるようになれば、理解が深まった証拠です：

1. **コードを読んで、何をしているか説明できる**
   - 他人に説明できるレベル

2. **エラーが出た時に、原因を推測できる**
   - エラーメッセージから問題箇所を特定できる

3. **小さな機能追加を、AIに頼らずにできる**
   - 既存のパターンを真似して実装できる

4. **「なぜそう書くのか」を説明できる**
   - 単に動くだけでなく、理由を理解している

5. **クラス設計を自分で考えられる**
   - どのクラスにどのメソッドを置くべきか判断できる

## 💡 よくある質問

**Q: AIに頼らずに作れないと、プログラマーとして失格ですか？**
A: いいえ。AIは「調べ物をする時間を短縮するツール」です。重要なのは：
- AIの回答を理解できるか
- なぜそのコードが動くのか説明できるか
- エラーを解決できるか

**Q: どのくらいの時間がかかりますか？**
A: 個人差がありますが、毎日1-2時間の学習で：
- レベル1-2: 2-3週間
- レベル3: 1-2ヶ月
- レベル4: 2-3ヶ月

**Q: 分からない時はどうすればいいですか？**
A: 
1. まず自分で考える（重要！）
2. エラーメッセージを読む
3. 公式ドキュメントを調べる
4. AIに「なぜ」を聞く（「どうすれば」ではなく）

**Q: オブジェクト指向が理解できません**
A: 最初は難しく感じるかもしれません。以下の順序で学習することをお勧めします：
1. まずクラスとオブジェクトの基本を理解
2. カプセル化（private、Getter/Setter）を理解
3. 継承を理解
4. ポリモーフィズムを理解
5. インターフェースを理解

焦らず、一つずつ確実に進めましょう。
